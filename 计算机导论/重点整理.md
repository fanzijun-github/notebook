### 154.7递归和迭代有何区别?所有的递归程序或算法都能转换为迭代程序或算法吗?所有的迭代程序或算法都能转换为递归程序或算法吗?请分别举例说明。
| 维度   | 递归（Recursion）       | 迭代（Iteration）                   |     |
| ---- | ------------------- | ------------------------------- | --- |
| 定义   | 函数直接或间接调用自身         | 通过循环结构（如 `for`、`while`）重复执行一段代码 |     |
| 控制机制 | 依赖函数调用栈（隐式栈）        | 依赖显式的循环变量和条件判断                  |     |
| 终止条件 | 基线条件（base case）     | 循环终止条件（如 `i < n`）               |     |
| 内存开销 | 每次调用产生新栈帧，空间复杂度通常更高 | 通常只用常量额外空间（除非循环内分配）             |     |
| 可读性  | 对某些问题（如树、分治）更直观自然   | 对简单重复任务更直接                      |     |
| 性能   | 函数调用有开销，可能栈溢出       | 通常更快、更节省内存                      |     |


| 问题         | 答案                   | PPT 依据                                  |
| ---------- | -------------------- | --------------------------------------- |
| 所有递归 → 迭代？ | ❌ 否（至少不是所有都能用常规迭代实现） | 第 37–40 页：阿克曼函数“用迭代不容易完成”，属于“执行也是递归的”类型 |
| 所有迭代 → 递归？ | ✅ 是                  | 第 41 页：阶乘的两种实现；递归是通用构造方法，迭代可视为递归的特例     |

### 189.6多级缓存、内存、外存之间有何区别
| 维度         | 多级缓存（Cache）               | 内存（RAM）            | 外存（如 SSD/HDD）          |
| ---------- | ------------------------- | ------------------ | ---------------------- |
| 物理位置       | CPU 芯片内部（L1/L2）或芯片附近（L3）  | 主板上的内存条            | 主机箱内的独立设备              |
| 存储介质       | SRAM（静态随机存取存储器）           | DRAM（动态随机存取存储器）    | NAND 闪存（SSD） / 磁盘（HDD） |
| 访问速度       | 极快（纳秒级，～1–10 ns）          | 快（几十纳秒，～50–100 ns） | 慢（SSD：微秒级；HDD：毫秒级）     |
| 容量         | 很小（L1: 几十KB；L3: 几MB～几十MB） | 中等（通常 8GB–64GB+）   | 很大（500GB–数TB）          |
| 成本（每GB）    | 极高                        | 中等                 | 低（尤其 HDD）              |
| 是否易失       | 是（断电丢失）                   | 是（断电丢失）            | 否（断电后数据保留）             |
| CPU 是否直接访问 | 是（最快路径）                   | 是（通过内存总线）          | 否（需先加载到内存）             |
| 用途         | 缓存频繁访问的指令和数据              | 存放正在运行的程序和数据       | 存放操作系统、应用、文件等          |

### 426.5IP 地址的结构是怎样的?IP 地址可以分为哪几类?子网掩码的作用是什么?
#### **IP 地址的结构（以 IPv4 为例）**

- 长度：**32 位二进制数**，通常用**点分十进制**表示，如 `192.168.1.1`
- 结构：由 **网络号（Network ID） + 主机号（Host ID）** 两部分组成
    - 网络号：标识所属网络
    - 主机号：标识该网络中的具体设备

| 类别  | 第一字节范围    | 网络号位数 | 主机号位数 | 网络数量     | 每网主机数    | 用途            |
| --- | --------- | ----- | ----- | -------- | -------- | ------------- |
| A 类 | 1 – 126   | 8 位   | 24 位  | 126 个    | 约 1677 万 | 大型网络（如政府、军方）  |
| B 类 | 128 – 191 | 16 位  | 16 位  | 16,384 个 | 约 6.5 万  | 中型网络（如大学、大企业） |
| C 类 | 192 – 223 | 24 位  | 8 位   | 209 万+   | 254 台    | 小型网络（如家庭、小公司） |
| D 类 | 224 – 239 | —     | —     | —        | —        | 组播（Multicast） |
| E 类 | 240 – 255 | —     | —     | —        | —        | 保留（实验用）       |
#### **子网掩码（Subnet Mask）的作用**

- **核心作用**：**区分 IP 地址中的网络号和主机号部分**。
- 表示方式：同样为 32 位，连续的 1 表示网络位，连续的 0 表示主机位。
    - 例如：`255.255.255.0`（即 `/24`）表示前 24 位是网络号，后 8 位是主机号。
- **如何使用**：
    - 将 IP 地址与子网掩码进行 **按位与（AND）运算**，得到**网络地址**。
    - 例：IP = `192.168.1.10`，Mask = `255.255.255.0` → 网络地址 = `192.168.1.0`
- **其他作用**：
    - 判断两个 IP 是否在同一子网（网络地址相同则在同一子网）；
    - 支持**子网划分**（Subnetting），提高 IP 地址利用率。
